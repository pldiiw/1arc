CHIP-8 interpreter CLI tool. General usage: ./cli.js [options] [suboptions] [subparameter] Options: -s, --state The file where the engine's state resides. (default: .engine_state.chip8.json) Subcommands: load [-b,--binary] [-d,--dry-run] Load program into a new CHIP-8 engine. cycle [-d,--dry-run] [amount] Run one or multiple cycle, i.e. read next instruction(s). display [-1,--pixel-on ] [-0,--pixel-off ] Print the engine's display. input [-n,--no] Press or not a key of the engine's keypad. inspect [-f,--format ] [-r,--range [start]-[end]] [component] Show engine's components bare data. edit Open the state file in your favorite editor to edit it safely. convert [-t,--to ] [-o,--output ] Little utility to convert CHIP-8 programs to raw binary or text. help [subcommand] Show the help.

convert()

Convert subcommand - Convert program files from/to binary to/from text. Usage: ./cli.js [options] convert [suboptions] source-file: The path to the file containing the CHIP-8 program to translate. Suboptions: -o, --output The file to same the converted program to. By default, outputs the program to stdout. -t, --to The format to convert the program file to. The possible values are: bin, binary, txt, text. Using one of the encoding implitcly indicates that the input program file is in the other format. Defaults to text (alias: txt).

Inspect subcommand - Show the bare data of the engine's various components. Usage: ./cli.js [options] inspect [suboptions] [component] component: The component of the engine to inspect, defaults to "all". Possible values: data, I, timer, sound, memory, pc, pointer, stack, display, keypad, all Options: -s, --state The file containing the engine's state to inspect. (default: .engine_state.chip8.json) Suboptions: -f, --format Change the base in which to display the engine's data, defaults to 10. -r, --range [start]-[end] When inspecting memory or the display, show only a portion of it. Defaults to "-", which is from 0 until the end.

load()

Load subcommand - Create a new CHIP-8 engine with the given source code fed into its memory and ready to be executed. Usage: ./cli.js [options] load [subptions] source-file: Path to the file containing the CHIP-8 code you want to load into the new engine. Options: -s, --state Save new engine to file, instead of default. (default: .engine_state.chip8.json) Suboptions: -b, --binary Indicate to the file parser that the source file specified is in a binary format, not text. -d, --dry-run Do not save the new engine to the state file. Can help to verify that a given CHIP-8 program is free of any syntax error.
